## Phase 1 — Dataset Collection & Problems Solved

### Problem 1: 6 Datasets, All Different Class Names & Indices

**Solution:** Built `remap_datasets.py` — custom per-dataset `class_map` dict remapping every dataset's indices to the unified schema above. Unwanted classes (boots, gloves, goggles, rope) deleted from labels.

### Problem 2: Missing Images in Dataset6
10 label files had no corresponding image.

**Solution:** Script's `find_image()` function skipped orphaned labels automatically with warnings.

### Problem 3: Duplicate Images
111 duplicate images found across datasets (all from dataset1, Roboflow augmentation copies).

**Solution:** MD5 hash-based deduplication in `dataset_utils.py`. Removed with `dry_run=False`.

### Problem 4: Class Balance
Checked with class balance report script. Result: **2.9x imbalance ratio** — healthy. No resampling needed.

**Final merged dataset:** `merged_dataset/` — **29,053 images**, 5 classes, existing train/val/test splits preserved.

## Why Shuffle Is Important For This Training

Your combined dataset has two very different sources being copied in order — first all 29k merged PPE images, then all 2,550 person images. **Without shuffle**, the training batches in early epochs would look like this:

- Batches 1–600: pure merged PPE data (helmet, vest, no-helmet, no-vest heavy)
- Last ~50 batches: pure person data

This creates a mini version of the same catastrophic forgetting problem you already experienced with the fine-tune. The model would oscillate between learning PPE and learning person instead of learning both simultaneously.

**With shuffle ON** (default), every batch is a random mix of both datasets — the model sees all 5 classes together in every epoch from the start.

**What did improve significantly:** `no-helmet` jumped from 0.878 → 0.918 (+3.9%). This is actually a meaningful safety-critical improvement — detecting workers without helmets is the most important class for real deployment.

**Overall mAP drop of -0.4%** is within noise, but the person regression is real.
